Jamstack

so let's start off talking defintions

"Jamstack"

  • Yet another marketing term
  • Previously known as JAMStack
  • Previously known as static sites
So yes, it's another marketing term, but it works well to set basic expectations Previously known as JAMStack with the J A M being an ancronym we will talk about later In the past, just known as static sites, but that term had problems because it implied a lack of interactivity

My definition

A way of building sites that relies on pre-rendered dynamic content living in production as static assets enhanced with client-side JavaScript and APIs for interactivity.

The "Old" Stack

  • Apache (web server)
  • ColdFusion (app server)
  • MySQL (database server)
lets take a step back and look at the "old" way of doing things, at least for me

Pros!

  • Very flexible
  • Very powerful
a LOT of power i wasn't actually using i had the benefit of free hosting, free licenses but obviously most people do not

"Static Sites"

  • Over ten years ago, first Static Site Generator (SSG) released
  • Dynamic app server in development
  • Output static files for production
lets take a step back

Example

  • Create some data (JSON, YAML, etc)
  • SSG "merges" this into HTML
  • Deploy

For Today

11ty

  • https://www.11ty.io
  • Open Source
  • Node based
  • Template Languages: HTML, Markdown, JavaScript, Nunjucks, Liquid, Handlebars, Mustache, EJS, Haml, Pug

Installation

npm install -g @11ty/eleventy

Commands (Some)

eleventy

eleventy --input=. --output=./_site

eleventy --serve

Demo

Show 11ty_one, a single file. Run eleventy to show the output, then eleventy serve. Demonstrate lack of auto reload Show 11ty_two, example of Liquid

Layouts

  • Specified in front matter (YAML, JSON, JS)
  • Loaded from /_includes
  • Can use any template language
  • Can be chained

Front Matter


---
layout: something
title: a value
something: another value
---

11ty_three - layout. front matter, show title, show link changing, show content

Working with Data

  • A way to define variables for your site
  • Front matter
  • File-based data (JSON or JS)
  • For one other file, for a directory, for a site (global)
  • For global data, stored in _data
